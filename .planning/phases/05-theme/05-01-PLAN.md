  <files>apps/site/server/core/theme/types.ts</files>
  <action>
Create the theme type definitions file following the pattern from `apps/site/server/core/plugin/types.ts`.

Define these types and interfaces:

1. **ThemeColors** interface:
   - `primary: string` (default: '#3B82F6')
   - `secondary: string` (default: '#64748B')
   - `accent: string` (default: '#8B5CF6')
   - `background: string` (default: '#FFFFFF')
   - `surface: string` (default: '#F8FAFC')
   - `text: string` (default: '#0F172A')
   - `textMuted: string` (default: '#64748B')

2. **ThemeFonts** interface:
   - `heading: string`
   - `body: string`
   - `mono: string`

3. **ThemeSpacing** interface:
   - `unit: string` (e.g. '8px')
   - `containerMax: string` (e.g. '720px')
   - `contentPadding: string` (e.g. '24px')

4. **ThemeBorderRadius** interface:
   - `small: string`
   - `medium: string`
   - `large: string`

5. **ThemeLayout** interface:
   - `headerPosition: 'top' | 'sticky' | 'hidden'`
   - `sidebarPosition: 'left' | 'right' | 'none'`
   - `footerStyle: 'simple' | 'detailed' | 'minimal'`

6. **ThemeConfig** interface:
   - `colors: ThemeColors`
   - `fonts: ThemeFonts`
   - `spacing: ThemeSpacing`
   - `borderRadius: ThemeBorderRadius`
   - `layout: ThemeLayout`

7. **ThemeMeta** interface:
   - `name: string`
   - `version: string`
   - `author?: string`
   - `description?: string`
   - `screenshot?: string`

8. **ThemeManifest** interface:
   - `meta: ThemeMeta`
   - `config: ThemeConfig`

9. **HookEvent** type:
   - `'theme:before-load' | 'theme:loaded' | 'theme:activated' | 'theme:changed' | 'theme:css-applied'`

10. **HookContext** interface:
    - `themeName: string`
    - `config?: ThemeConfig`
    - `cssVariables?: string`
    - `timestamp: number`

11. **HookHandler** type:
    - `(event: HookEvent, context: HookContext) => void | Promise<void>`

12. **CSSVariablesMap** function:
    - Input: `ThemeConfig`
    - Output: `Record<string, string>` mapping CSS variable names to values
    - Maps: colors -> `--color-{key}`, fonts -> `--font-{key}`, spacing -> `--spacing-{key}`, borderRadius -> `--radius-{key}`, layout -> `--layout-{key}`
    - Include `--container-max` and `--content-padding` from spacing

All types should be exported. Use JSDoc comments following the plugin types.ts pattern.

<task type="auto">
  <name>Task 2: Create hook system EventEmitter</name>
  <files>apps/site/server/core/hooks/event-emitter.ts, apps/site/server/core/hooks/index.ts</files>
  <action>
Create the hook system as a simple event emitter pattern.

**event-emitter.ts:**

Implement `EventEmitter` class with:
- `private handlers: Map<string, Set<HookHandler>>` — stores registered handlers by event name
- `on(event: HookEvent, handler: HookHandler): () => void` — register handler, returns unsubscribe function
- `once(event: HookEvent, handler: HookHandler): () => void` — register handler that auto-removes after first call
- `off(event: HookEvent, handler: HookHandler): void` — remove specific handler
- `async dispatch(event: HookEvent, context: HookContext): Promise<void>` — call all handlers for event in registration order, await each, catch and log errors without stopping other handlers
- `clear(): void` — remove all handlers (for testing/reset)

Import `HookEvent`, `HookContext`, `HookHandler` from `../theme/types`.

Create and export singleton: `export const hookEmitter = new EventEmitter()`

**index.ts:**

Barrel export:
```typescript
export { EventEmitter, hookEmitter } from './event-emitter'
export type { HookEvent, HookContext, HookHandler } from '../theme/types'
```

The event emitter should be lightweight — no priority system, no middleware, just on/off/dispatch. Follow the singleton pattern from `apps/site/server/core/plugin/manager.ts` (export class + export singleton instance).
