Create the ThemeManager class following the PluginManager pattern from `apps/site/server/core/plugin/manager.ts`.

**manager.ts:**

Implement `ThemeManager` class with:

1. **Private state:**
   - `themes: Map<string, ThemeManifest>` — registered themes
   - `activeTheme: string | null` — currently active theme name
   - `themesDir: string` — path to themes directory, resolve using `process.cwd()` + `'themes'`

2. **`scanThemes(): void`** — Synchronous method to discover themes:
   - Read `themesDir` using `fs.readdirSync`
   - For each subdirectory, check if `config.json` exists
   - If exists, read and parse `config.json`, validate it has required fields (colors, fonts, spacing, borderRadius, layout)
   - Store as `ThemeManifest` in `this.themes` map
   - Log: `[ThemeManager] Discovered {count} theme(s)`

3. **`loadTheme(themeName: string): ThemeManifest | null`** — Load a specific theme:
   - Check if already in `this.themes` map
   - If not, read from `themesDir/{themeName}/config.json`
   - Parse and validate
   - Store in map
   - Dispatch `'theme:loaded'` hook event
   - Return manifest or null if not found

4. **`async activate(themeName: string): Promise<void>`** — Activate a theme:
   - Load theme if not already loaded
   - Throw if theme not found
   - Deactivate current active theme in DB (set `isActive: false`)
   - Update new theme as active in DB (upsert pattern from PluginManager.persistToDb)
   - Set `this.activeTheme = themeName`
   - Dispatch `'theme:activated'` hook event

5. **`getActive(): ThemeManifest | null`** — Get currently active theme:
   - If `activeTheme` is null, return null
   - Return `this.themes.get(activeTheme)` or null

6. **`getActiveCSS(): string`** — Generate CSS Variables text for active theme:
   - Get active theme config
   - If null, return empty string
   - Call `CSSVariablesMap(config)` to get variable map
   - Build CSS text: `:root { {each variable as --name: value; } }`
   - Return CSS text

7. **`getThemeCSS(themeName: string): string`** — Generate CSS for a specific theme:
   - Same as getActiveCSS but for specified theme

8. **`getAll(): ThemeManifest[]`** — Return all registered themes

9. **`getConfig(themeName: string): ThemeConfig | undefined`** — Get config for a theme

10. **`async loadFromDb(): Promise<void>`** — Load active theme from database:
    - Query `themeSettings` table for `isActive: true`
    - If found, set `this.activeTheme` and load theme
    - If not found, activate 'default' theme (call `this.activate('default')`)
    - Log: `[ThemeManager] Active theme: {name}`

11. **`private validateConfig(config: unknown): config is ThemeConfig`** — Validate config shape:
    - Check required top-level keys: colors, fonts, spacing, borderRadius, layout
    - Check required color keys: primary, secondary, accent, background, surface, text, textMuted
    - Return boolean

12. **`private async persistToDb(themeName: string, config: ThemeConfig): Promise<void>`** — Upsert to themeSettings:
    - Check if row exists for themeName
    - If exists: update config, isActive, updatedAt
    - If not: insert new row
    - Follow the exact upsert pattern from PluginManager.persistToDb
