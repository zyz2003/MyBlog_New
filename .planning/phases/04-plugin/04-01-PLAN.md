    - getPluginsForMountPoint(mountPoint) returns enabled plugins that declare that mount point

Create `apps/site/server/core/plugin/manager.ts` implementing the PluginManager class.

Key implementation details:
1. Private fields: `plugins` (Map<string, PluginAdapter>), `enabledPlugins` (Set<string>), `configs` (Map<string, Record<string, any>>)
2. `register(plugin)`: Check for duplicate name, store in map
3. `enable(pluginName, config)`: Look up plugin, validate config against schema (check required fields, run validators), call onInit for extended plugins, add to enabled set and config map
4. `disable(pluginName)`: Look up plugin, call onUnmount for extended plugins, remove from enabled set and config map
5. `getEnabled(type?)`: Filter enabled plugins by optional type
6. `getPluginsForMountPoint(mountPoint)`: Filter enabled plugins that include the mount point
7. `getConfig(pluginName)`: Return config from map
8. `updateConfig(pluginName, newConfig)`: Validate, call onConfigChange for extended plugins, update config map
9. Type guard `isExtended(plugin)`: Check for extended interface methods
10. Export singleton `pluginManager = new PluginManager()`
