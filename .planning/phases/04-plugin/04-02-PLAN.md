  <action>
Create `apps/site/components/plugins/PluginMount.vue` - the core component that renders a single plugin.

Props:
- `pluginName: string` - plugin identifier
- `componentPath?: string` - path to Vue component (for component-based plugins)
- `scriptUrl?: string` - script URL (for script-injection plugins)
- `config: Record<string, any>` - plugin configuration

Implementation:
1. If `componentPath` is provided:
   - Use `defineAsyncComponent(() => import(componentPath))` to lazy-load the Vue component
   - Render with `<component :is="PluginComponent" :config="config" />`
   - This is SSR-safe - renders on server

2. If `scriptUrl` is provided (no componentPath):
   - Wrap in `<ClientOnly>` to ensure client-only execution
   - Use a `ref<HTMLElement>` for the container
   - In `onMounted`, create a `<script>` tag with the src, append to container
   - In `onUnmounted`, remove the script tag

3. Handle loading state with Suspense or v-if

4. Use `<script setup lang="ts">` syntax per project conventions

Template structure:
```vue
<template>
  <div :data-plugin="pluginName" class="plugin-mount">
    <!-- Component-based plugin (SSR safe) -->
    <Suspense v-if="componentPath">
      <component :is="PluginComponent" :config="config" />
      <template #fallback>
        <div class="plugin-loading" />
      </template>
    </Suspense>

    <!-- Script-injection plugin (client only) -->
    <ClientOnly v-else-if="scriptUrl">
      <div ref="scriptContainer" class="plugin-script-container" />
    </ClientOnly>
  </div>
</template>
```

Create `apps/site/components/plugins/PluginRenderer.vue` - fetches and renders all plugins for a given mount point.

Props:
- `mountPoint: MountPoint` - which mount point to render
- `postId?: string` - optional post ID for post-specific plugins (like comments)

Implementation:
1. Use `useFetch('/api/plugins/enabled')` to get all enabled plugins
2. Filter plugins by mountPoint
3. Render each plugin using PluginMount component
4. Pass postId to plugins that need it (e.g., comment plugins)

Template:
```vue
<template>
  <div :data-mount-point="mountPoint" class="plugin-renderer">
    <PluginMount
      v-for="plugin in mountPlugins"
      :key="plugin.name"
      :plugin-name="plugin.name"
      :component-path="plugin.componentPath"
      :script-url="plugin.scriptUrl"
      :config="plugin.config"
    />
  </div>
</template>
```

Then create 6 mount point wrapper components. Each is a thin wrapper around PluginRenderer:
