API endpoints:
- GET /api/categories -> returns flat category list (CategoryService.list)
- GET /api/categories/tree -> returns nested tree (CategoryService.tree)
- POST /api/categories -> create category (body: CategoryCreateInput)
- PUT /api/categories/:id -> update category (body: CategoryUpdateInput)
- DELETE /api/categories/:id -> delete category
- GET /api/tags -> returns all tags
- POST /api/tags -> create tag (body: TagCreateInput)
- PUT /api/tags/:id -> update tag (body: TagUpdateInput)
- DELETE /api/tags/:id -> delete tag

  <files>apps/site/pages/admin/categories.vue, apps/site/components/admin/categories/CategoryTree.vue, apps/site/components/admin/categories/CategoryForm.vue</files>
  <behavior>
    - CategoryTree.vue renders categories recursively; each node shows name, slug badge, child count; expand/collapse toggle for nodes with children; edit and delete action buttons per node; delete button disabled (or confirm blocked) if node has children
    - CategoryForm.vue is a modal dialog with fields: name (required), slug (required, auto-generated from name), description (textarea), parent (select dropdown of all categories excluding self and descendants when editing), sortOrder (number input); emits submit with form data and close event
    - categories.vue fetches tree from GET /api/categories/tree on mount; renders CategoryTree; has "New Category" button that opens CategoryForm in create mode; edit action opens CategoryForm in edit mode with pre-filled data; create calls POST /api/categories; update calls PUT /api/categories/:id; delete calls DELETE /api/categories/:id with confirm dialog; refreshes tree after each mutation; shows loading skeleton on initial load; shows empty state when no categories exist
  </behavior>
  <action>
    Follow the established pattern from apps/site/pages/admin/articles/index.vue:
    - Use `definePageMeta({ layout: 'admin' })` at top of script setup
    - Use `const api = useAdminApi()` for all API calls
    - Use UnoCSS utility classes matching the existing admin style (card, btn-primary, gray-* colors, rounded-md, shadow-sm, etc.)
    - Use i-heroicons-* icons matching Sidebar conventions

    CategoryTree.vue:
    - Recursive component accepting `categories: CategoryTreeNode[]` prop and `level: number` (default 0) for indentation
    - Emit 'edit' with category id and 'delete' with category id
    - Use CSS padding-left or ml-* for tree depth visualization
    - Show expand/collapse chevron icon that rotates; nodes without children show no chevron
    - Display child count badge next to category name

    CategoryForm.vue:
    - Accept props: `category` (null for create, populated for edit), `allCategories` (flat list for parent select)
    - Use a Teleport to body for the modal overlay
    - Auto-generate slug from name using a simple slugify: lowercase, replace spaces with hyphens, remove special chars
    - When editing, exclude the category itself and its descendants from the parent dropdown (prevent circular refs)
    - Validate: name required, slug required, slug must be URL-safe
    - Emit 'submit' with form data, 'close' to dismiss

    categories.vue:
    - Fetch tree on mount via api.get('/api/categories/tree')
    - Track editingCategory ref (null or category object for edit mode)
    - Track showForm ref for modal visibility
    - On create: open empty form, on submit call api.post('/api/categories', data), refresh tree
    - On edit: open form with category data, on submit call api.put(`/api/categories/${id}`, data), refresh tree
    - On delete: confirm dialog, call api.del(`/api/categories/${id}`), refresh tree
    - Show error alerts on API failures using try/catch
  </action>
