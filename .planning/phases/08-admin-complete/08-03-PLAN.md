    - settings.vue fetches all settings from GET /api/settings on mount; organizes into tabs: Site, SEO, Reading, Social; each tab renders SettingsForm with appropriate fields and current values; "Save" button calls PUT /api/settings with all changed items; shows success toast/alert after save; shows loading state on initial load
  </behavior>

    SettingsForm.vue:
    - Props: `fields` (array of { key: string, label: string, type: 'text'|'textarea'|'number'|'boolean', placeholder?: string }), `modelValue` (Record<string, any>)
    - For text/textarea: standard input/textarea with v-model bound to modelValue[field.key]
    - For number: input type="number" with min/max if provided
    - For boolean: toggle switch (styled checkbox)
    - Emit 'update:modelValue' on any change (v-model pattern)
    - Layout: label on left, input on right, stacked vertically with gap

    - definePageMeta({ layout: 'admin' })

    - Define tab structure:
      ```
      tabs = [
        { id: 'site', label: 'Site', icon: 'i-heroicons-globe-alt' },
        { id: 'seo', label: 'SEO', icon: 'i-heroicons-magnifying-glass' },
        { id: 'reading', label: 'Reading', icon: 'i-heroicons-book-open' },
      ```

      - social: github (text), twitter (text), weibo (text), email (text)
    - Track activeTab ref, settingsValues ref, changedKeys ref (Set of keys that were modified)
    - On field change: update settingsValues, add key to changedKeys
    - On save: build items array from changedKeys, call api.put('/api/settings', { items }), clear changedKeys, show success message

    - Save button: fixed at bottom or in header, disabled when changedKeys is empty
