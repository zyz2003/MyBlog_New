# Phase 00 Plan 02: Phase 2 核心框架测试审查

## Objective

审查 Phase 2 (核心框架) 的测试文件，确保测试基于规范而非实现。

**Purpose:** 发现并修复"测试迁就正式代码"的问题

**Output:** 审查报告，修复的问题列表

---

## Must-Haves

### Truths

- 插件系统测试必须基于 REQUIREMENTS.md 中的插件需求
- 主题系统测试必须基于 REQUIREMENTS.md 中的主题需求
- 测试失败时必须修复正式代码，不是测试文件

## Tasks

### Task 1: Audit plugin system tests

**Behavior:**

- 审查 `packages/plugin/src/__tests__/*.test.ts`
- 验证测试依据是插件系统需求而非当前实现
- 识别"测试迁就"问题

**Action:**

1. Read all plugin test files
2. Compare test assertions against REQUIREMENTS.md plugin requirements
3. Identify cases where tests were written to match buggy implementation
4. Document findings in audit report

**Verify:**

- Check if 16 plugin hooks are tested
- Check if plugin lifecycle is tested
- Check if tests would fail if implementation is wrong

---

### Task 2: Audit theme system tests

**Behavior:**

- 审查 `packages/core/theme/__tests__/theme-manager.test.ts`
- 验证 CSS Variables 切换测试完整性
- 验证主题配置文件解析测试

**Action:**

1. Read theme-manager test file
2. Compare against REQUIREMENTS.md theme requirements
3. Identify missing tests or "accommodation" tests
4. Document findings

**Verify:**

- CSS Variables switching is tested
- Theme config parsing is tested
- Theme preview is tested
