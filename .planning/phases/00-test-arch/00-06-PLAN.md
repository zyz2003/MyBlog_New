# Phase 00 Plan 06: 测试架构约束与后续阶段规范

建立测试架构约束和后续阶段 (Phase 5-11) 的测试规范，杜绝"测试迁就"问题。

## Tasks

### Task 1: Create test architecture constraints document

**Behavior:**

- 定义四层测试架构
- 定义每层测试的依据
- 定义违反处理流程

**Action:**
Create `.planning/phases/00-test-arch/TEST-ARCH-CONSTRAINTS.md` with:

```
Layer 1: Contract Verification (TypeScript, Schema)
Layer 2: Unit Tests (isolated functions/classes)
Layer 3: Integration Tests (module collaboration)
Layer 4: E2E Tests (user workflows)

For each layer:
- What it verifies
- Source of truth
- What to fix when tests fail
```

---

### Task 2: Create test templates

**Behavior:**

- 提供单元测试模板
- 提供集成测试模板
- 提供 API 契约测试模板

**Action:**
Create:

- `apps/site/tests/unit.template.test.ts`
- `apps/site/tests/integration.template.test.ts`
- `apps/site/tests/api-contract.template.test.ts`

Each template includes:

- Standard describe structure
- Common test patterns
- Comments explaining what to base tests on

---

### Task 3: Update CI/CD workflow

**Behavior:**

- 在 CI/CD 中添加四层验证流程
- 确保 Schema 验证集成到 CI

**Action:**
Update `.github/workflows/ci.yml` with:

```yaml
jobs:
  contract-verification:
    # Type check, Schema verification, Lint
  unit-tests:
    # Fast, isolated unit tests
  integration-tests:
    # Module collaboration tests
  e2e-tests:
    # Full user workflow tests (Phase 5+)
```
